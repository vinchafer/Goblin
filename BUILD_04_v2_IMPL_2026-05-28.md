## 10. Surprising

**Two divergences from pixel-perfect v2 — neither fixable within the locked edit scope. Flagged, not acted on.**

1. **Top-anchoring (v2 change #1) is NOT achieved by these edits alone — HIGH.** `EmptyChat.tsx`'s own outer wrapper still carries `flex: 1`, `justifyContent: 'center'`, `padding: '40px 24px'` (L28-33). When the parent gets `.chat-scroll--empty` (`justify-content: flex-start; padding-top: 120px`), the EmptyChat child's `flex: 1` grows it to fill the full column height and its `justifyContent: center` re-centers the block vertically — negating the 120px top-anchor. Result: the empty block renders roughly centered, not in the upper third. Matching v2 requires a follow-up edit to EmptyChat's outer wrapper (remove `flex: 1` + `justifyContent: 'center'`, drop its `padding-top` to avoid double-padding with the parent's 120px). That wrapper was explicitly out of EDIT 1/2 scope (text + variant only), so it was left untouched pending approval.

2. **`align-items: center` now applies to the message list (non-empty state) — MEDIUM.** The shared scroll div previously had no `align-items` (default `stretch`); `.chat-scroll` introduces `align-items: center`, which will horizontally center/shrink message rows unless Message.tsx sets `width:100%`/`align-self:stretch` on its rows. The non-empty surface is out of this pass's scope (Message.tsx do-not-touch). Needs a visual check on a populated chat before shipping; if messages squash, the base class needs `align-items` reconsidered or the message rows need `align-self: stretch`.

3. Minor: padding changed `20px 16px` → `24px` on the scroll area (per mockup base). Intended, noted for completeness.
