## The gap (worth founder attention ‚Äî a beta blocker for the headline flow)
"Send to Code" on a raw fenced HTML code block creates an **unsaved `html-snippet` editor
buffer**. After **Apply Changes**, the editor shows the snippet (`editors=1`) **but the Files
panel still reads "No files yet"** ‚Äî the content was **not persisted as a named project file**
(e.g. `index.html`). Build then correctly refuses: *"Project has no files to deploy."*

So the simplest happy path ‚Äî *chat one HTML file ‚Üí Send to Code ‚Üí Apply ‚Üí Build* ‚Äî does **not**
yield a deployable file without an extra (missing/!undiscovered) save-as-file step. The model was
asked for "one complete HTML file" and produced valid HTML, but Goblin treats it as a snippet,
not a file.

This is distinct from the **Sprint-2 services-level proof**, which deployed successfully (HTTP 200
in ~14s) ‚Äî because that path wrote real files via the API. The *UI* path for a raw snippet stops
short of a persisted file.
