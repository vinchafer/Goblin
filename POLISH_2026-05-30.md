## 4 — Settings typography + spacing pass

**Systemic fix — `.helper-text` class** (the prior-pass `.settings-section p →
16px` rule had enlarged intentionally-small helpers on desktop).

`globals.css`, inside `@container settings-pane (min-width: 600px)`:

```css
.settings-section .helper-text {
  font-size: var(--t-small-fs) !important;  /* 14px */
  color: var(--ink-2) !important;
}
```

Scope note: the rule (like the body-`p` rule it counters) lives in the desktop
`settings-pane` container only. On the mobile sheet, helpers keep their authored
inline size — the "too large" problem was desktop-only.

**Applied `className="helper-text"` to:**

|---|---|---|---|
| AboutPage | Version/commit line | 13px `--text-meta` (div) | 14px `--ink-2` |

## 5 — Dark mode contrast audit

**Dark mode IS active site-wide.** `lib/theme.tsx` defaults to `'system'` and
sets `data-theme="dark"` on `<html>`; the `[data-theme="dark"]` block in
`design-tokens.css` flips the ink/surface ramps app-wide. So a user on a dark OS
gets the dark theme everywhere — the founder's complaints are real.

Root cause: elements using `--text-faint` (= `--ink-disabled`, `#5A523D` in
dark) on the dark sidebar surface, and an active-tab pill bound to a token that
flips.

| Offender | File | Was | Now |
|---|---|---|---|
| PROJEKTE/CHATS eyebrows | Sidebar.tsx | `--text-faint` (#5A523D dark) | `--ink-3` (#968768 dark, AA) |
| Time-ago (project + chat) | Sidebar.tsx, RecentChatRow.tsx | `--text-faint` | `--ink-3` |
| "See all chats →" | Sidebar.tsx | `--text-faint` | `--ink-3` |
| **Active tab label** | Header.tsx | bg `--surface-1` (→ #133224 dark) on text `--brand-header` #1A3A2A = invisible | bg `--paper` (LOCKED #FBF7EC, never flips) |

The header is always brand-green; inactive tab labels are `--ink-on-dark-2`
(#D8CBA8) — already ≥7:1, left unchanged. The real header bug was the **active**
pill: `--surface-1` flips to dark green in dark mode, so dark-green text sat on a
dark-green pill. `--paper` is a LOCKED anchor that does not flip → readable in
both themes. `--ink-3` resolves to `#74694F` (light) / `#968768` (dark) — both
pass for UI text.

  in dark mode" — the inactive labels (`--ink-on-dark-2`) were actually fine.
  The real bug was the *active* pill binding to `--surface-1`, which flips to

  `--ink-3`; the code actually used `--text-faint` (= `--ink-disabled`,
  `#5A523D` in dark) — even fainter. Moved to `--ink-3` (the dark-safe tertiary).
- **Item 8 label was never missing.** The brief speculated a stripped label; the
  label was present. The defect was purely the `--ink-deep` (near-black)
  background vs the spec's `--brand-green`.
- **"komplett dunkel" fix is global by nature.** `.gobl-btn.primary` is shared,
  so the brand-green correction lands on every dashboard primary CTA — a net
  consistency win, but worth noting it's not project-page-local.
