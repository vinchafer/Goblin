Kannst du mir eine Stripe-Checkout-Komponente für meine Next.js-App bauen? Mit dem aktuellen App Router.

Klar. Hier eine minimale Server-Route für Stripe Checkout mit dem App Router. Sie erstellt eine Checkout-Session und gibt die url zurück.

app/api/checkout/route.ts
import Stripe from 'stripe';
import { NextResponse } from 'next/server';

const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!);

export async function POST(req: Request) {
  const { priceId } = await req.json();
  const session = await stripe.checkout.sessions.create({
    mode: 'payment',
    line_items: [{ price: priceId, quantity: 1 }],
    success_url: `${process.env.URL}/success`,
    cancel_url: `${process.env.URL}/cancel`,
  });
  return NextResponse.json({ url: session.url });
}

Leg die Route unter app/api/checkout/route.ts ab. Du brauchst noch:

  • STRIPE_SECRET_KEY in .env.local
  • eine priceId aus dem Stripe-Dashboard
  • Erfolgs- und Abbruch-URLs (siehe Stripe Docs)

Viewport 1: Screen 05 (Active Chat) mit Code-Block VARIANT A (Paper / hell).
Viewport 2: Screen 05 mit Code-Block VARIANT D (Ink-Deep / dunkel) — gleicher Chat, nur Treatment anders.

→ V1 ↔ V2 direkt vergleichen, um zwischen Paper und Ink-Deep für das finale Code-Block-Treatment zu wählen.

Shiki-Theme (für späteren Code-Build): VARIANT A = helles Theme (z.B. vitesse-light, hier manuell auf gedämpfte Erdtöne gemappt: Keyword=Brand Green, String=warmes Burgundy #8B4A3A, Number/Type=Gold-700). VARIANT D = vitesse-dark (Keyword Pink-Red, String Hellblau, Function Lila) — liest sauber auf Ink-Deep.
Neue Tokens nötig: keine. Alle Werte aus design-tokens.css. Einzige Nicht-Token-Literale: die Shiki-Highlight-Farben (Theme-eigene Syntaxpalette, kein Brand-Token — String-Burgundy #8B4A3A für A; #F97583/#9ECBFF/#FFAB70/#B392F0/#79B8FF für D).
Inhalts-Hinweis: 06 zeigt user1 → asst1 (komplett, mit Code) → user2 → asst2 streaming. user2 ist hier gesendet, damit die streamende Antwort einen Bezug hat (kohärenter Generating-State).
