## Fixed this pass
1. **`meta-viewport` (moderate, all 12 routes)** — root layout `viewport` had
   `maximumScale: 1, userScalable: false`, which blocks pinch-zoom (WCAG 1.4.4). Removed both.
   One global fix → cleared on every route. *(`apps/web/app/layout.tsx`)*
2. **Legal footer contrast (serious, /terms /imprint)** — footer text + nav links were
   `rgba(255,255,255,0.5)` on `--brand-green` (fails 4.5:1). Raised to `rgba(255,255,255,0.72)`.
   Isolated inline values, not tokens. *(`apps/web/app/(legal)/layout.tsx`)*
3. **`link-in-text-block` (serious, /imprint)** — the EU ODR URL inside body text relied on color
   alone. Added `text-decoration: underline`. *(`apps/web/app/(legal)/imprint/page.tsx`)*

Result: **/terms, /imprint, /help are serious-free.** `/help` had no serious findings to begin
with; its remaining items are moderate (`landmark-one-main`, `region`).

## Residual serious — `color-contrast` on locked v1.1 tokens (DEFERRED to founder)
All remaining serious findings are contrast, and every one traces to a token or color decision in

**Recommended (founder approval — these edit locked tokens):**
- Darken `--ink-3` ~10–12% (e.g. → ~`#6a5f47`) so captions clear 4.5:1 on sand surfaces — this
  single token change resolves the majority of remaining findings.
- Darken `--text-faint` for empty-state copy (or only use it on `--surface-0`, not white).
- For the gold wordmark: bump to ≥18.66px or darken the gold one step on green.
- Landing decorative/code-mockup text: confirm intent — if these are deliberately low-emphasis,
  consider an explicit AA-exempt decision; otherwise nudge their opacities.
