
(Replaced with an explanatory comment. `--ink-4` kept — dashboard-specific, no global equivalent.)

**Header token swaps** (`Header.tsx`):
| line | element | before → after |
|------|---------|----------------|
| 105 | hamburger icon | `rgba(255,255,255,0.85)` → `var(--ink-on-dark-1)` |
| 143 | crumb separator "/" | `rgba(255,255,255,0.3)` → `var(--ink-on-dark-3)` |

| 266 | inactive tab pill | `rgba(244,236,216,0.78)` → `var(--ink-on-dark-2)` |

A2.5 confirms `--ink-on-dark-1` on `--brand-green` = 12.4:1 AAA.

**Two deliberate deviations from EDIT 1.3 (flagged):**
1. **Wordmark kept `--brand-gold`** (not `--ink-on-dark-1`). Reason: gold-on-green is A2.5-safe (5.6:1 AA for large UI text), `built_04_v2.html` shows the wordmark gold (rule #5 mockup-is-truth), and a gold mark + cream wordmark would be internally inconsistent. The g-mark stays gold; the wordmark matches it.
2. **Active tab pill kept `--brand-header` (green)** (not `--ink-on-dark-1`). The active pill has a **white** (`--surface-1`) background — cream text on white would be invisible. Active pill must be dark-on-light. Only the *inactive* pills (on the green bar) were swapped to `--ink-on-dark-2`.

## 3. Workstream 2 — Sidebar (partial)

**Data-flow fix (DONE) — the "Builder"/"B" bug.**
Before: `Sidebar` read `userName`/`userEmail` from props `DashboardShell` never passed → always `'Builder'`/`'B'`.
```
- interface SidebarProps { ... userEmail?: string; userName?: string; ... }
- export function Sidebar({ ..., userEmail, userName, ... })
- const displayName = userName || userEmail?.split('@')[0] || 'Builder';
+ import { useUser } from '@/lib/hooks/useUser';
+ const { fullName, email, plan } = useUser();
+ const displayName = fullName || email?.split('@')[0] || 'Builder';
```
Now the sidebar reads the same source as the header avatar — real initial + name, no more "Builder"/"B" mismatch.

**Top label (DONE):** removed the stray `{displayName}` text in the sidebar header (the "Builder" beside the collapse toggle). Kept the `Goblin.` wordmark + collapse toggle (mockup-true; EDIT 2.2's literal "toggle only" would also drop the wordmark, which the 03/04 mockups keep — flagged).

**Plan badge (DONE):** user-pill now shows `{plan.name}` ("Build"/"Pro"/"Power") as a small badge between name and gear.

**DEFERRED (needs a focused pass):** collapse behavior (260↔48px + localStorage + main-area reflow), projects content-fit height, chats max-5 + "See all", toggle-icon swap (`PanelLeftClose`/`Open`), full token sweep. Reason: a full rewrite of the 524-line component (chat fetch, mobile overlay variant, `RecentChatRow`/`SidebarUsage` integration) is its own build-risk unit.

## 4. Workstream 3 — AvatarMenu desktop popover (DONE)

Before: always `<BottomSheet>` (centered iOS sheet on desktop). After: viewport-branched.
- **Desktop (≥768px):** `DesktopMenuPopover` — `createPortal` to `document.body`, `position:fixed` anchored to the avatar's `getBoundingClientRect()` (`top: rect.bottom+8`, right-aligned to the trigger), width 240, `--surface-0` bg, `--rule` border, `--shadow-2`, `--radius-md`. Click-outside (mousedown) + Escape close; focus moves into the menu on open + Tab focus-trap. Pattern mirrors `ComposerPlusPopover` (read-only reference, unchanged).
- **Mobile (<768px):** unchanged `BottomSheet`.
- Trigger: `aria-haspopup="menu"` + `aria-expanded`. Same `SettingsCard`/dense `SettingsRow` body in both shells (no enlargement on desktop) → items stay at `--t-small-fs`.
- Items unchanged: Einstellungen, Plan upgraden, Hilfe, Routing (toggle, keeps GitBranch icon), Abmelden (`--rust`). No leading icons except Routing.

Note: "Einstellungen" still calls `setShowSettingsSheet(true)` (opens the sheet) on both viewports — the desktop branch will swap to `SettingsModal` once WS4 lands.
