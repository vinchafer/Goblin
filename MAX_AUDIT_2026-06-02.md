# Max Audit — 2026-06-02

*Persona: Max, 31, Berlin, Content Manager. Has Claude Pro + ChatGPT Plus, hits limits, wants to build a newsletter landing page without touching Cursor/VSCode. Mobile-first (S-Bahn, iPhone). Low tolerance for jargon, medium for friction.*

## Max's story (3 paragraphs)

I land on the page on my phone and honestly it speaks my language for about ten seconds — "Tell it what you want. It ships." Yes, that's the dream, I don't want to open a code editor. I tap "Start building free," it says no credit card, good. I sign up. Then the app is suddenly in **German** ("Sag Goblin, was du bauen willst") even though the homepage was English — mildly disorienting but fine, I'm German, I'll take it. The dashboard is calm and I find the big "Sag Goblin was du bauen willst" box. I type *"Ich will eine Landing-Page für meinen Newsletter bauen."*

And then nothing builds. I get a red error — something about **"Model not found in LiteLLM."** I don't know what LiteLLM is. I don't know what a model is. I just know the box where I typed my idea gave me an error in tech-language. I try once more, same thing. This is exactly the moment the prompt warned about: **"will try twice, then leave."** I'm at the leave point. The one thing I came for — describe it, get a thing — didn't happen, and the error didn't tell me what *I* should do about it.

If I'd gotten past that, the rest actually looks made-for-me: there's a clear "Veröffentlichen" button with a confirm step (I like that it doesn't just YOLO-deploy — I'd be scared of that), and after publishing there's a **Live-URL I can copy and send to my girlfriend** — that's the feature I actually want. The file thing ("Dateien → Explorer") is more than I need but not in my face. On mobile it all stacks into one column and I can read it without pinch-zooming. So the *shell* is built for me. The *engine* told me to go away.

## Findings

**CRITICAL — "Model not found in LiteLLM" on my first real prompt.** *Max's reaction:* "Ich würde hier sofort aufgeben. Ich hab eine Idee getippt und es kommt ein Fehler mit Wörtern die ich nicht kenne." *Do:* a vibe-coder must never see "LiteLLM" or "model." If setup is needed, walk me through connecting a key in plain words *before* I can type a doomed prompt — or just give trials a working model.

**SERIOUS — dev jargon in errors and labels.** "Model", "LiteLLM", "BYOK", "Secrets", "Deploy/Veröffentlichen" mix. *Reaction:* "Was ist BYOK? Was sind Secrets?" *Do:* plain-German microcopy ("Dein KI-Schlüssel", "Online stellen"), and never surface internal component names in errors.

**SERIOUS — EN homepage → DE app.** *Reaction:* "Kurz dachte ich, ich bin auf der falschen Seite." *Do:* match the language I arrived in.

**MEDIUM — the KEYBOARD SHORTCUTS overlay.** On mobile/desktop it sits over the corner of the screen. *Reaction:* "Was ist ⌘K? Ich hab gar keine Tastatur am Handy." *Do:* hide keyboard-shortcut hints on touch devices.

**POSITIVE — the Save→Publish gap (Zwischenraum) is exactly right for me.** I get to publish on purpose, with a confirm, not by accident. Sprint-6 instinct confirmed intact.

**POSITIVE — persistent Live-URL + Copy.** This is *the* feature I'd come back for: build → publish → copy link → send it. It no longer disappears after a second.

**POSITIVE — mobile.** No tiny desktop UI crammed onto a phone; cards stack, text is readable, no horizontal scroll. The dashboard remembers my recent chats/sessions so "where was I" is answered.

## Verdict

**Would Max use Goblin instead of Claude + copy-paste?** Today: **no** — because the first prompt errored and I bounced before building anything. The cruel irony is that everything *after* a successful generation is built for me (publish-on-purpose, copyable live URL, mobile-friendly, remembers my work). The product is one fix away from being the thing I want. **The instant a trial prompt actually produces a page, Max stays.** Until then, Max types one idea, gets "LiteLLM", and goes back to pasting code from ChatGPT.
