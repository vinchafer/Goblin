## Environment constraint discovered (Phase 0)
- Local web (`localhost:3000`) → **prod Railway API** (`.env.local`: `goblinapi-production.up.railway.app`) → **prod Supabase**.
- Browser client calls from `localhost` are **CORS-limited** to the prod API (dashboard "Keine Verbindung zum Server" confirms). The Code-Tab `/api/code-sessions` probe also fails from localhost → CodeWorkspace falls back to the classic single-buffer editor.
- **Consequence:** new code runs only on localhost, but localhost can't reach the API. So **UI** is verifiable locally; **DB-persist / API-streaming E2E** requires founder to push + deploy, then verify on prod (`goblin-web.vercel.app`). Migrations 0057/0058 are NOT applied (founder applies).
- **Mitigation:** every new feature degrades gracefully pre-migration (defensive best-effort writes) and uses a localStorage hint so the UI is correct even before the DB column exists.
