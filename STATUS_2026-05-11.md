# GOBLIN — Project Status 2026-05-11

## Was ist Goblin?

**AI-powered Code Studio** — User beschreiben was sie bauen wollen, Goblin generiert Code.
Live URL: https://justgoblin.com  
API URL: https://goblinapi-production.up.railway.app

BYOK-first: User bringen ihre eigenen API Keys (Anthropic, OpenAI, Google, Groq, DeepSeek, Mistral, xAI, Together, Fireworks). Goblin routet Requests durch diese Keys. Kein LLM-Anbieter-Lock-in.

## AI / Model Routing

### 3 Layer Architektur

```
Layer 1: Goblin Hosted (coming soon — env: GOBLIN_GPU_ENDPOINT)
Layer 2: Free API Pool (company keys, kostenlos für User)
Layer 3: BYOK (User's own keys)
```

### Free API Pool (konfiguriert via env)
- `GOOGLE_FREE_API_KEY` → Gemini 2.0 Flash
- `GROQ_FREE_API_KEY` → Llama 3.3 70B  
- `CEREBRAS_FREE_API_KEY` → Llama 3.3 70B (Cerebras)
- `OPENROUTER_FREE_API_KEY` → DeepSeek Chat

### BYOK Provider Priority (auto-select ohne explizites Modell)
anthropic → openai → deepseek → groq → mistral → google → xai → together

### Unterstützte Provider
anthropic, openai, google, groq, mistral, xai, deepseek, together, fireworks, openrouter, custom

### Model-Calls
- Anthropic: via `@anthropic-ai/sdk` (nativer SDK, nicht OpenAI-compat)
- Alle anderen: via `openai` SDK (OpenAI-compatible)
- Optional: via LiteLLM-Proxy (`LITELLM_BASE_URL`) für alle Provider

### Key Validation beim Speichern
Jeder BYOK-Key wird vor dem Speichern validiert:
- Anthropic: `POST /v1/messages` mit claude-3-5-haiku, max_tokens=1
- OpenAI: `GET /v1/models`
- Alle anderen: `GET /v1/models` oder Provider-äquivalent
- Timeout: 5000ms

## Architektur-Entscheidungen (wichtig für neue Features)

- **Keine relativen API-Calls** — immer `NEXT_PUBLIC_API_URL` explizit verwenden
- **BYOK-Keys werden serverseitig validiert** — nie raw Key ans Frontend zurückgeben
- **storage_path pattern**: `projects/{projectId}/{filePath}` in Hetzner S3
- **In-Memory Fallback** für File Storage wenn S3 nicht konfiguriert (kein Crash)
- **Startup Migration Runner** (`startup-migrations.ts`) prüft Schema aber führt keine DDL aus — nur Warnungen
- **Model Cache** 60s per User — bei Key-Änderungen invalidieren via `DELETE /api/models/cache`
- **SSE für alle Long-Running Operations** (Chat, Generate, Deploy) — nie WebSockets
