## 3. AKTUELLER ASSET-LAGE-CHECK

### 3.1 Differentiation Assets (was Goblin von Cursor/Lovable/Cline unterscheidet)

**Mobile-First** (USP):
- Echte Mobile-Web-App, nicht nur Mobile-Responsive
- Send-to-Code Pattern für autonome Builds vom Handy
- Bottom-Sheet-Navigation statt Desktop-Sidebar-Adaption
- Long-Press-Context-Sheets mit Haptic Feedback
**Konkurrenz-Vergleich:** Cursor + Cline = Desktop-Only. Lovable + Bolt = Browser-Only (kein Mobile-First-Design). **Goblin's Vorteil: nativer Mobile-Workflow.**

**Model Intelligence Layer** (Asset aus 9R):
- 4 aktive Public-Benchmark-Sources kombiniert (OpenRouter, Aider, HuggingFace, SWE-Bench)
- 978 Modelle indexiert, 2630 Rankings
- Auto-Refresh alle 6h
- User sieht in Echtzeit welches LLM für seinen Task aktuell am besten ist
- Composer integriert EMPFOHLEN-Badges
**Konkurrenz-Vergleich:** Cursor + Cline = manueller Model-Picker ohne Empfehlung. Lovable + Bolt = hardcoded auf bestimmte Provider. **Goblin's Vorteil: Investor-Story "Wir routen intelligent basierend auf 5 Public-Benchmarks, automatisch täglich aktualisiert".**

**BYOK-First mit User-Education** (USP nach Session 13):
- Onboarding leitet User zum eigenen Free-Tier-Key in 30 Sekunden
- Keine Lock-In auf Goblin-Token-Pool
- User behält Kontrolle über Spend
**Konkurrenz-Vergleich:** Lovable + Bolt = Monatlicher Goblin-Token-Pool. Cursor + Cline = BYOK aber kein Onboarding-Help. **Goblin's Vorteil: einziges Tool das User aktiv zum Free-Tier-Key hilft.**

**ToS-Compliance als Wettbewerbsvorteil**:
- Bewusste Entscheidung gegen LMArena/Artificial-Analysis-Scraping
- Keine "Free-Pool"-Strategie über Goblin-Accounts
- Saubere Provider-Auswahl in Adapter-Liste
**Was das wert ist:** Bei späterem Enterprise-Sales oder Investor-Tech-DD ein klares Differenzierungsmerkmal — Goblin baut auf Sand-festem Fundament, nicht auf grauer Zone.

- pre-existing TSC errors in billing.ts + cache.ts gefixt

- Pro User Key Encryption Key (KEK) generiert bei Signup

- User-BYOK-Keys (Data Encryption Keys, DEK) werden mit User's KEK verschlüsselt

- Migration `0042_byok_per_user_encryption.sql`

- QR-Code-Setup-Flow in Settings → Sicherheit
- Recovery-Codes (10 Stück, einmal anzeigen)
- Account-Lockout nach 5 failed login attempts (15 min cooldown via Redis oder DB-Table)
- Session-Timeout nach 24h Inaktivität (Supabase Session-Refresh-Logic)
- Active Sessions UI in Settings → Sicherheit:
  - Liste aller aktiven Sessions mit Device + IP + Last-Used
  - "Logout this device" Button per Session
  - "Logout all other devices" Button global
- Email-Notification bei Login von neuem Device (via Resend)
- Migration `0043_2fa_and_sessions.sql`
